# Cowork or Claude Code for this task?

**Recommendation: default to Claude Code once you're set up. Reach for Cowork
when you want zero friction, or when you're away from your own machine.**

They're the same assistant with a different amount of fence around it. The
choice is really "how much freedom does this task need, and how much setup am I
willing to spend?"

## Use Cowork when

- You haven't set up Claude Code yet, or you're on a machine that isn't yours.
- The task lives inside your files and connected apps — read this doc, organize
  this folder, draft from these sources.
- You want it to just work, with a sandbox making mistakes safe by default.

## Use Claude Code when

- The task needs the fence gone: installing a tool, calling an arbitrary web
  service, logging into a command-line tool, editing your Google Docs for real.
- You want it to run for a long time, or several agents at once.
- It's something you'll repeat — Claude Code's `CLAUDE.md` and skills make it
  stick.

## The honest summary

Cowork is the softer landing and a genuinely good tool; Claude Code is where
most people end up for real work because nothing is fenced off. If you've done
the [Part 2](../02-claude-code.md) setup, let Claude Code be your default and
keep Cowork for the quick, from-anywhere jobs.
